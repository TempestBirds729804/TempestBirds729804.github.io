<Leetcode Hot 100>: 3.连续最长序列

Created2025-03-12|Updated2025-04-11|LeetcodeHot100
|Post Views:

LeetcodeHot100哈希表（HashMap）题目其三

1. 题目概述

给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

示例 1：

输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。

示例 2：

输入：nums = [0,3,7,2,5,8,4,6,0,1]
输出：9

示例 3：

输入：nums = [1,0,1,2]
输出：3

提示：

  • 0 <= nums.length <= 105
  • -109 <= nums[i] <= 109

2. 解题思路

这一道题目的要求是要求出连续的最长序列。（这个序列本身是乱序的）

所以说，这道题第一眼看上去，笔者肯定是想到了排序，但是这个题目给了要求复杂度为O(n) ,显然排序的话就需要O(nlogn)了，故不能用这招。那有什么办法可以在遍历一次的情况下对最长连续序列去计数呢？

朴素的想法显然就是在单次遍历的过程中，当拿到起始数字时候，开始计数，直到这个序列结束为止，然后再去重复这个过程对下一段连续值的长度计数，并且始终把这个计数值维持成最大数值，直到遍历结束为止。

那么朴素的想法有了，在落实的时候显而易见引出了两个关于细节的问题：其一是如何判断一段连续数值的起始点？；其二是在计数的过程中如何去查下一个元素到底在不在集合里？（因为用常规手段遍历去查找的话时间复杂度肯定还是超了）

综上两点，我们很容易就想到了这道题目的适用算法——HashMap

2.1 Python题解

class Solution(object):
    def longestConsecutive(self, nums):
        """
        :type nums: List[int]
        :rtype: int
        """
        if not nums:
            return 0

        num_set = set(nums)
        max_length = 0

        for num in num_set:
            if num-1 not in num_set:
                current_num = num
                current_length = 1

                while current_num+1 in num_set:
                    current_num+=1
                    current_length+=1

                max_length = max(max_length,current_length)
        return max_length

2.2 C++题解

#include <unordered_set>
#include <vector>
using namespace std;


class Solution{
public:
     int longestConsecutive(vector<int>& nums){
        unordered_set<int> numSet(nums.begin(),nums.end());
        int maxLen=0;
        for (int num: numSet){
            //从“序列的起点开始”
            if(int num : numSet){
                int currentNum=num;
                int length=1;
                while(numSet.count(currentNum+1)){
                    currentNum++;
                    length++;
                }
                maxLen=max(maxLen,length);
            }
         }
        return maxLen;
    }
}

2.3 思路详解

我们使用一个 哈希集合（set） 来快速判断某个数字是否存在。我们只在每个序列的起点处开始遍历整个连续序列。

比如：

  • 如果我们遇到 100，我们看 99 在不在集合里？如果不在，说明 100 是一个新序列的开始。
  • 然后再找 101, 102…直到不连续为止。

这样我们每个元素最多只访问一次，由于使用了HashMap查找，显而易见时间复杂度就是 O(n)。

以上就是本道题目的解题思路，感谢观看！
