<Leetcode Hot 100>: 2.字符异位词分类

Created2025-03-07|Updated2025-03-12|LeetcodeHot100
|Post Views:

LeetcodeHot100哈希表（HashMap）题目其二

1. 题目概述

给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

字母异位词 是由重新排列源单词的所有字母得到的一个新单词。

示例 1:

输入: strs = ["eat", "tea", "tan", "ate", "nat", "bat"]
输出: [[“bat”],[“nat”,”tan”],[“ate”,”eat”,”tea”]]

示例 2:

输入: strs = [""]
输出: [[“”]]

示例 3:

输入: strs = ["a"]
输出: [[“a”]]

提示：

  • 1 <= strs.length <= 104
  • 0 <= strs[i].length <= 100
  • strs[i] 仅包含小写字母

2. 解题思路

这道题的核心是字母异位词的分组，我们需要找到相同单词不同顺序的特征。

我们显而易见的是，组成单词的字母是一样的，它们只是顺序不一样的，所以就需要找到一种方法，把这些异位词映射到HASHMAP的同一个Key上面，这样也就轻松实现了分组。

2.1 排序法

这个方法的思路：遍历字符串列表strs的每一个word，然后把里边的拿到的word用python里面的sorted方法按照字母排序成一个新词组，排序之后的词组可以作为HASHMAP(defaultdict)的key来存储，显而易见这样异位词对应的Key就是相同的，所以也就实现了分组。

代码示例：

from collections import defaultdict

def group_anagrams(strs):
    anagrams = defaultdict(list)  # 创建一个哈希表，默认值是空列表

    for word in strs:
        sorted_word = ''.join(sorted(word))  # 将单词按字母排序
        anagrams[sorted_word].append(word)  # 按排序后的单词作为key进行存储

    return list(anagrams.values())  # 返回所有分组

所以这个解法的时空复杂度如何呢？

时间复杂度：

假设 n 是字符串个数，k 是字符串的最大长度：排序单词 需要 O(k log k) 时间。遍历 n 个单词，总时间是 O(n * k log k)。

所以 总时间复杂度是：O(n⋅klogk) ，这个复杂度可以接受

空间复杂度:

我们使用了一个 哈希表 来存储分组，键的数量最多是 n，每个键的值是一个列表，整体空间复杂度 O(nk)。

因此 总空间复杂度是：O(n⋅k)

Tip（因为贴主刚开始学Python，对刚遇到的语法相关的东西做个简单记录，不感兴趣可以跳过）：这里用了Python的collections库中的defaultdict类，它和普通的Python字典dict的区别是，后者在访问一个不存在的key的时候会直接抛出来一个keyError异常，所以在访问dict的value之前，需要手动检查对应的key是不是存在的，或者使用.get()的方法来提供默认值。

而对于defaultdict来说，就没有上述问题了，当访问一个不存在的key的时候，defaultdict会自动创建一个默认的值，而不是抛给你keyError。

它的基本语法如下：

from collections import defaultdict
default_dict = defaultdict(default_factory)

其中，default_factory是一个可以调用的对象（函数），用来给不存在的key提供一个默认值。常见用法有：

    • deffaultdict(list) → 默认值是 []
    • defaultdict(int) → 默认值是 0
    • defaultdict(set) → 默认值是 set()
    • defaultdict(lambda): 初始值 → 自定义默认值
特性 dict defaultdict

适合场景 需要手动检查key是否存在的场景 需要频繁地给新key赋默认值的情况
代码的简洁性 偏具体 更简略

2.2 使用字符计数的方法

对于排序方法，排序的时间复杂度是 O(k log k)。如果 k 很大（如 100），排序会比较慢。所以考虑进一步优化的情况下，可以用 字符计数法 来替代排序：

思路就是使用Counter或者数组来统计每一个单词的字符频次，这个指标显然也可以用来做异位词分类，而且还不用排序。用元组（a,b,c,…z）作为key，异位词的key显然就是相同的，这样就直接分组了。

代码实现如下：

def group_anagrams(strs):
    anagrams = defaultdict(list)

    for word in strs:
        count = [0] * 26  # 用26长度的数组存储字母出现次数
        for char in word:
            count[ord(char) - ord('a')] += 1  # 计算字符频次
        anagrams[tuple(count)].append(word)  # 用元组作为key

    return list(anagrams.values())

从时间复杂度的角度来看，统计字符频次：O(k)，遍历 n 个单词，总复杂度 O(nk)。所以比 O(nk log k) 更快一些，适用于 k 很大时。
